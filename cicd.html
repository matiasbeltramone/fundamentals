🚀 CI/CD con GitHub Actions

Integración y despliegue continuo explicado para principiantes

📌 Introducción

CI/CD significa Integración Continua y Entrega Continua. Nos ayuda a automatizar pruebas y despliegues para tener software siempre listo.

⚙️ Workflow básico


name: Deploy a GitHub Pages
on:
  push:
    branches: [ "main" ]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/configure-pages@v3
      - uses: actions/upload-pages-artifact@v1
        with:
          path: .
      - uses: actions/deploy-pages@v1

✅ Conclusión

Con GitHub Actions + Pages, cada vez que subís código a main, tu sitio se despliega automáticamente.
